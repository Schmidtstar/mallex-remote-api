Änderungen:
	1.	MenuScreen.tsx
	•	Ersetze die jetzige Header-Buttonleiste durch eine Tabs-Config (Array), die die sichtbaren Reiter definiert und Content via activeTab rendert.
	•	Tabs in dieser Reihenfolge: settings, profile, tasks, suggest, admin (+ optional rules, about).
	•	Admin nur anzeigen, wenn isAdmin === true oder localAdmin === true (Flag aus AuthContext bzw. local storage — es gibt bereits einen Local-Admin-Badge in i18n).
	•	Entferne sämtlichen alten Account-Form-Code (bereits geschehen); Logout bleibt ausschließlich im ProfileTab.
	•	In TasksTab: nutze weiterhin categories & challenges (unified), zeige links das icon aus categories.ts neben dem Label.
	2.	Neu: tabs/SuggestTab.tsx
	•	Formular: Select (Kategorie aus categories), Textarea (Aufgabe), Button „Senden“.
	•	Beim Senden: TaskSuggestionsContext.add({categoryId, text, createdAt}).
	•	Zeige nach Submit i18n-Thanks-Hinweis (kein Reload, Felder leeren).
	3.	Optional: tabs/RulesTab.tsx, tabs/AboutTab.tsx
	•	Simple statische Texte aus i18n; später erweiterbar.
	4.	de.json / en.json
	•	Unter menu.tabs ergänzen (falls noch nicht vorhanden):
	•	rules („Spielregeln“ / „Rules“)
	•	about („Über MALLEX“ / „About MALLEX“)
	•	ranklist („Spieler-Rangliste“ / „Player Ranking“) – nur Placeholder, Tab vorerst nicht aktiv verdrahten.
	•	Unter menu.suggest sicherstellen: title, choose, placeholder, submit, thanks sind vorhanden (bei dir schon da ✔️).

Akzeptanzkriterien:
	•	Menü zeigt Reiter: Einstellungen, Profil, Aufgaben, Vorschlagen (immer), Admin (nur bei Admin/LocalAdmin).
	•	Aufgaben listet pro Kategorie die Challenges aus derselben Datenquelle (keine Duplikate, keine Konsole-Warnings).
	•	Vorschlagen sendet Einträge an TaskSuggestionsContext und zeigt „Danke“-Hinweis; keine Network-Calls.
	•	Logout existiert nur im Profil-Tab.
	•	Keine Änderungen an Build/Runtime/Packages.
